# هيكلية نظام الشات العادي (Standard Chat System)

يحتوي نظام الشات على عدة طبقات برمجية موزعة بين المكونات (Components)، إدارة الحالة (State Management)، الخدمات (Services)، والوسائط (Assets). إليك الهيكلية الكاملة:

## 1. منطق الميزة (src/features/chat)
هذا هو المجلد الرئيسي الذي يحتوي على منطق الشات، وهو متبع لنمط `Feature-Sliced Design`.

```text
src/features/chat/
┣ 📂 components/               # المكونات الرسومية الخاصة بالشات
┃ ┣ 📂 layout/                 # الهيكل الرئيسي لواجهة الشات
┃ ┃ ┣ 📜 ChatLayout.tsx        # المكون الكلي الذي يجمع القائمة والمحادثة
┃ ┃ ┣ 📜 ChatSidebar.tsx       # قائمة المحادثات (Sidebar)
┃ ┃ ┗ 📜 ConversationArea.tsx  # منطقة المحادثة النشطة (Messages area)
┃ ┣ 📂 molecules/              # مكونات متوسطة الحجم
┃ ┃ ┣ 📜 ChatInput.tsx         # شريط الإدخال (النص، الصور، التسجيل، الإيموجي)
┃ ┃ ┣ 📜 MessageBubble.tsx     # فقاعة الرسالة (التشفير، التفاعلات، الحالة)
┃ ┃ ┣ 📜 ChatHeader.tsx        # رأس المحادثة (اسم المستخدم، التشفير)
┃ ┃ ┣ 📜 DecryptedImage.tsx    # معالجة وعرض الصور المشفرة
┃ ┃ ┗ 📜 TypingIndicator.tsx   # مؤشر "يكتب الآن..."
┃ ┗ 📂 security/               # مكونات متعلقة بالأمان والتشفير
┃   ┣ 📜 EncryptionKeyDialog.tsx # عرض مفتاح التشفير الخاص بالمستخدم
┃   ┗ 📜 DeviceFingerprint.tsx  # بصمة الجهاز لزيادة الأمان
┣ 📂 hooks/                    # المنطق البرمجي (التعامل مع Supabase)
┃ ┣ 📜 useChat.tsx             # جلب المحادثات والرسائل والاشتراك في التغييرات
┃ ┣ 📜 useChatSound.ts         # إدارة أصوات التنبيهات (الإرسال/الاستقبال)
┃ ┣ 📜 useE2EE.tsx             # إدارة التشفير طرف-لطرف (Keys management)
┃ ┣ 📜 useSendMessage.ts       # منطق إرسال جميع أنواع الرسائل (نص، صورة، صوت)
┃ ┗ 📜 useVideoChat.ts         # (اختياري) الربط مع نظام الفيديو (موجود في نفس المجلد)
┣ 📂 services/                 # خدمات تقنية مساعدة
┃ ┣ 📜 crypto.ts               # المحرك الأساسي للتشفير (AES/RSA)
┃ ┗ 📜 geoFingerprint.ts       # منطق تحديد الموقع والبصمة للأمان
┣ 📂 store/                    # إدارة الحالة (Zustand)
┃ ┗ 📜 chatStore.ts            # مخزن الحالة (المحادثة النشطة، الرسائل، الحظر)
┣ 📂 constants/                # الثوابت
┃ ┗ 📜 sounds.ts               # بيانات الأصوات بصيغة Base64 (النقرة، الإرسال)
┣ 📜 ChatProvider.tsx          # المزود الرئيسي لسياق الشات (React Context)
┗ 📜 index.ts                  # ملف التصدير الرئيسي للميزة
```

## 2. المكونات المشتركة والخدمات العامة (src/)
توجد بعض الملفات خارج مجلد `features` لكنها أساسية لعمل الشات:

```text
src/
┣ 📂 components/ui/             # مكونات واجهة المستخدم العامة
┃ ┣ 📜 EmojiPicker.tsx         # منتقي الإيموجي المتطور (Apple-style)
┃ ┗ 📜 EmojiText.tsx           # عرض النصوص مع تحويل الإيموجي لصور
┣ 📂 utils/                     # أدوات مساعدة
┃ ┗ 📜 emoji.utils.ts          # معالجة وفلترة الإيموجي وتحويل الكود
┗ 📂 lib/                       # المكتبات الخارجية
  ┗ 📜 supabaseClient.ts       # إعدادات الاتصال بقاعدة البيانات (Realtime)
```

## 3. الوسائط والأصول (public/)
توجد الصور والأصوات الثابتة في المجلد العام لسهولة الوصول:

```text
public/
┣ 📂 assets/
┃ ┗ 📂 sounds/                 # ملفات الأصوات التفاعلية (mp3)
┃   ┣ 📜 capture.mp3
┃   ┣ 📜 check.mp3
┃   ┗ 📜 move.mp3
┣ 📂 emoji/                    # مجموعة ضخمة من صور الإيموجي (Apple v15)
┃ ┗ 📜 (آلاف ملفات png بأسماء unicode مثل 1f600.png)
┣ 📜 wtf.wav                   # صوت تنبيه خاص (صوت مضحك)
┣ 📜 girl_chat.webp            # صور توضيحية لواجهة الشات
┗ 📜 guy_chat.webp
```

---
### ملاحظات تقنية هامة:
1. **لا يوجد مجلد `api`:** منطق البيانات مدمج مباشرة في الـ `Hooks` باستخدام مكتبة `supabase-js`.
2. **التشفير:** ملف `crypto.ts` هو القلب النابض للنظام، حيث يقوم بتشفير الرسائل قبل إرسالها للسيرفر.
3. **الأصوات:** يتم استهلاك الأصوات بطريقتين: إما من ملف `constants/sounds.ts` (كبيانات Base64 فورية) أو من `public/assets/sounds` (كمانفات خارجية).
4. **الإيموجي:** يتم عرض الإيموجي كصور Apple حقيقية لضمان شكل موحد على جميع الأجهزة، ويتم ذلك عبر `EmojiText.tsx`.
